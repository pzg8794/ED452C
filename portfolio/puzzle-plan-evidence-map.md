# Puzzle Plan Evidence Map

## Purpose

This map preserves how the Session 2 ED452C discussion post can later become part of the broader teaching portfolio.

## Portfolio claim supported

Inclusive teaching requires designing access, participation, and assessment from the start rather than treating accommodations as after-the-fact fixes.

## Evidence pieces connected

### ED452C Session 2 reading

- Strengths-based view of disability.
- Inclusion as meaningful access and participation, not only classroom placement.
- Instruction should begin from student capacities, supports, and high expectations.

### Pine Brook IGNITE / Minecraft Education placement

- Grade 4 Minecraft Education students varied in keyboard fluency, reading confidence, executive functioning, prior Minecraft experience, and comfort with open-ended digital tasks.
- Startup routines, sign-in supports, projected click paths, vocabulary supports, debugging checkpoints, visual/oral directions, partner roles, and multiple response modes functioned as inclusive Tier 1 design.
- Coding competence could be hidden by language load, navigation friction, or interface confusion.

### EDU442 AI learning partner project

- AI use was framed as an equity dilemma involving disability/neurodivergence, language load, technology access, privacy, and student authorship.
- The classroom routine Ask -> Check -> Choose -> Change -> Explain supported student agency and visible thinking.

### ED452B / EDE498C Minecraft unit work

- ED452B served as the inclusive-design backbone for the Minecraft innovative unit.
- EDE498C strengthened the unit with real placement evidence from the TeachingPlacement trail.
- The unit used MTSS/UDL logic: support structures were embedded in Tier 1 instruction before individual failure.

### ED400 comprehensive portfolio

- ED400 frames the portfolio as a collage of coursework, practicum evidence, and reflections.
- ED452C posts should be saved as small, reusable reflection artifacts that can later support InTASC-aligned proficiency claims.
