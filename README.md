# ED452C Course Workspace

Piter Garcia | Instructional Strategies for Inclusive Classrooms C | Summer A 2026

This repository is the versioned workspace for ED452C course materials, discussion work, assignment planning, and portfolio-ready evidence. It follows the same portfolio logic used across the broader teacher-preparation repositories: course artifacts are not isolated one-off submissions; they are puzzle pieces that can later be assembled into a coherent certification portfolio.

The documentation is written with a public-safe discipline regardless of repository visibility: no raw student records, no identifiable placement details, and no Blackboard-only content copied beyond the student's own drafts and notes.

## Start here

1. `docs/course-map.md` - map of current ED452C materials and related repositories.
2. `docs/discussion-log.md` - running tracker of discussion posts created for Blackboard.
3. `docs/source-map.md` - source map for the teaching placement and related ED course repositories used for enrichment.
4. `discussions/session-02-discussion-post.md` - Session 2 discussion post, ready to paste into Blackboard.
5. `portfolio/puzzle-plan-evidence-map.md` - how this week's work connects to the larger portfolio puzzle plan.
6. `docs/work-log.md` - brief record of work completed and decisions made.

## Public-safety rules

- Do not commit raw student work, identifiable student information, private school documents, or Blackboard-only course content that should not be public.
- Prefer de-identified summaries, reflection notes, and source maps over raw placement records.
- When using placement evidence, describe patterns and instructional implications rather than naming students.
- Keep discussion posts in `discussions/` and log every created post in `docs/discussion-log.md`.
- Keep portfolio-facing synthesis in `portfolio/` so later ED400 work can reuse it cleanly.
- Keep source provenance in `docs/source-map.md`, especially when a discussion post draws from private placement records or another ED course repository.

## Working principle

The core ED452C theme is inclusive instructional design. For this portfolio, that means every discussion post and assignment should answer two questions:

1. What does the reading or course session say about inclusive practice?
2. Where do I see that idea in my actual teaching placements, especially Pine Brook IGNITE, Minecraft Education, AI-supported learning, and differentiated computer science instruction?

The puzzle-plan habit is simple: every post should preserve a small, reusable connection between a course idea, a placement pattern, and a later portfolio claim.
